Fix: Duplicate Phone Number Detection Not Working
Context
Nombor telefon yang sama muncul berkali-kali dalam Lead Report tapi tidak dikesan sebagai duplicate. Contoh: Jakarta Bandung (60168217133) dan Pakej Balkan 8 Negara (60175189798) ada 2 rekod dengan nombor sama tapi is_duplicate = false.

Punca masalah: Duplicate detection hanya check dalam batch import semasa sahaja (guna Set<string> dalam memory). Tidak check terhadap rekod sedia ada dalam database. Manual add lead (POST) juga langsung tiada duplicate check.

Plan
1. Create shared phone utility (src/lib/phone-utils.ts)
Extract cleanPhoneNumber dari import-reports/route.ts ke shared module. Tambah:

fetchExistingPhones(supabase) — fetch semua no_phone dari DB ke Set (untuk bulk import)
checkPhoneExistsInDB(supabase, phone, excludeId?) — check single phone (untuk manual add/edit)
2. Fix import duplicate detection (src/app/api/import-reports/route.ts)
Both paths (file upload + Google Sheets):

Sebelum loop import, call fetchExistingPhones() untuk load semua nombor sedia ada
Check duplicate = dalam batch ATAU dalam database
Affects lines ~329 (file upload) and ~560 (Google Sheets)
3. Fix manual lead creation (src/app/api/lead-reports/route.ts POST)
Normalize phone number dengan cleanPhoneNumber
Check if phone exists in DB via checkPhoneExistsInDB
Set is_duplicate accordingly
4. Fix lead update (src/app/api/lead-reports/route.ts PUT)
Bila user tukar no_phone, re-check duplicate status
Guna excludeId supaya tak match dengan diri sendiri
5. Create retroactive scan endpoint (src/app/api/lead-reports/scan-duplicates/route.ts)
Scan SEMUA rekod dalam database
Order by date_lead ASC, created_at ASC — rekod pertama = original, selebihnya = duplicate
Cross-staff detection (scan semua staff sekaligus)
Batch update is_duplicate untuk semua rekod
Also normalize stored phone numbers yang belum normalized
Superadmin only
6. Add "Scan Duplicate" button (src/components/LeadReportTab.tsx)
Button untuk trigger retroactive scan
Show result: berapa duplicate ditemui daripada berapa rekod
Auto-refresh table selepas scan

Verification
Run retroactive scan — semua duplicate dalam DB akan dikesan dan ditandakan
Import baru — nombor yang dah ada dalam DB akan auto-detect sebagai duplicate
Manual add lead — nombor sedia ada akan ditandakan duplicate
Edit lead tukar nombor — duplicate status dikemaskini
